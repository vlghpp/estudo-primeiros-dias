Neste dia foi ensinado sobre a importância do git, de criar um repositório remoto no github através do git. Também foi ensinado alguns comandos

  • git init -> para iniciar um repositório em uma pasta específica
  • git remote add 'origin' 'link do repositório do github' -> para linkar o seu repositório local com o repositório no github e fazer as modificações pelo cmd
  • git add -> adiciona alguma pasta/arquivo/documento que queira para Staging Area
  • git commit -m 'comentário' -> para adicionar um commit, ou seja, alguma modificação no código, uma nova versão.
  • git push origin 'nome da branch' -> para 'empurrar' tudo que foi comitado para o repositório na nuvem, ou seja, o repositório criado no github

'nome da branch' -> para acessar o nome da branch você da git status, la vai aparecer 'On branch ...'

**Origin -> O "origin" é um nome convencional frequentemente usado para se referir ao repositório remoto principal, mas você poderia escolher outro nome se preferisse..
